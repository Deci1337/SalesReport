# SalesReport

SalesReport — это настольное WPF-приложение для формирования отчётов отдела продаж электронной техники.

## Описание

Приложение позволяет:

- Просматривать отчёты о продажах ноутбуков, смартфонов и планшетов
- Фильтровать отчёты по типу устройства и периоду (день / неделя / месяц / квартал / год)
- Отслеживать динамику цены конкретной модели на графике
- Сохранять сводный отчёт в текстовый файл
- Хранить данные в формате JSON или XML (с возможностью переключения)

## Основные возможности

- Выбор периода через календарь и выпадающий список
- Список отчётов с чекбоксами — активируется автоматически по выбранной дате
- Сводный отчёт из нескольких источников без дублирования устройств
- Сортировка таблицы по артикулу (по возрастанию / убыванию)
- График динамики цены: ось X — дата продажи, ось Y — средняя цена
- Переключение формата хранения JSON ↔ XML с автоматическим копированием данных
- Сохранение отчёта в файл вида `Отчет_№1_за_01.05-31.05.txt`

## Технологии

- C#, WPF (Windows Presentation Foundation)
- .NET Framework 4.7.2
- Newtonsoft.Json — сериализация в JSON
- System.Xml.Serialization — сериализация в XML
- WPF Canvas — отрисовка графика (без внешних библиотек)

## Структура проекта

Бизнес-логика (классы устройств, отчёты, сериализация) вынесена в отдельную библиотеку `Model` с папками `Core` и `Data`.

Пользовательский интерфейс находится в проекте `SalesReportApp` и разделён на три окна: главное меню `MainWindow`, окно отчёта `ReportWindow` и график цены `PriceChartWindow`.

## Как запустить

1. Клонируйте репозиторий или откройте папку проекта
2. Откройте `SalesReportApp\SalesReportApp.sln` в Visual Studio 2022
3. Правой кнопкой по solution → **Restore NuGet Packages**
4. **Ctrl+Shift+B** — собрать решение
5. **F5** — запустить

При первом запуске автоматически создаётся папка `data_json/` с тестовыми данными (27 устройств, 9 отчётов за март–май 2026).

## Как пользоваться

### Главное окно

| Элемент | Назначение |
|---|---|
| Тип устройства | Фильтр: Все / Ноутбуки / Смартфоны / Планшеты |
| Период отчёта | День / Неделя / Месяц / Квартал / Год |
| Дата | Выбор даты через календарь |
| Список отчётов | Появляется после выбора даты и периода; отмечай нужные отчёты чекбоксами |
| Формат файлов | JSON или XML; при смене данные копируются в новый формат |
| Показать отчёт | Активна когда отмечен хотя бы один отчёт |

### Окно отчёта

| Элемент | Назначение |
|---|---|
| ↑ / ↓ По артикулу | Сортировка таблицы |
| Таблица | Артикул, бренд, модель, тип, цена, дата продажи |
| Комбобокс артикула | Выбор модели для графика |
| Динамика цены | Активна при выбранном артикуле; открывает график |
| Сохранить отчёт | Сохраняет таблицу в `.txt` файл рядом с `.exe` |

### График динамики цены

Открывается в отдельном окне. Показывает как менялась средняя цена продажи выбранной модели по датам. Каждая точка — отдельная продажа.

## Важные замечания

- Каждое конкретное устройство (по ID) продаётся ровно один раз и находится только в одном отчёте
- Одна модель (артикул) может продаваться несколько раз — это разные физические экземпляры
- Тестовые данные пересоздаются только если папка `data_json/` пуста или отсутствует
